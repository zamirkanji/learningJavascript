How JS Works

  • Memory Heap
    • Memory Leak
  • Call Stack
    • First in, last out
  • Single-Threaded that can be non-blocking
    • Only one call-stack, can do one thing at a time
  • Synchronous Programming
    • Line one gets executed, then two, then three
    • Each line can't start until the previous one finishes
  • Asynchronous Programming
    • Callback functions get run in the background so other functions can run
  • Stack Overflow (Overflowing the stack from recursion)
